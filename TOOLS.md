# Debugging Tools in Visual Studio 2022

| Feature                                                                                             | Beschreibung                                                                                                                  |
|---------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------|
| [Run to Click](https://devblogs.microsoft.com/devops/run-to-click-debugging-in-visual-studio-2017/)     | Führe den Code bis zur gewünschten Zeile aus, indem du einfach auf das grüne Symbol neben der Zeile klickst – kein Breakpoint nötig. |
| [Force Run to Cursor/Click](https://devblogs.microsoft.com/visualstudio/debug-with-force-run-to-cursor/) | Erzwinge das Ausführen bis zum Cursor, auch wenn Ausnahmen oder Breakpoints dazwischen liegen würden.                         |
| [Angeheftete DataTips](https://docs.microsoft.com/en-us/visualstudio/debugger/view-data-values-in-data-tips-in-the-code-editor?view=vs-2019) | Werte von Variablen dauerhaft im Editor anzeigen lassen, um sie beim Debuggen im Blick zu behalten.                          |
| [Setze als nächste Anweisung](https://docs.microsoft.com/en-us/visualstudio/debugger/navigating-through-code-with-the-debugger?view=vs-2019#BKMK_Set_the_next_statement_to_execute) | Bestimme, welche Codezeile als nächstes ausgeführt werden soll, ohne das Programm neu zu starten.                            |
| [DebuggerDisplay](https://devblogs.microsoft.com/visualstudio/customize-object-displays-in-the-visual-studio-debugger-your-way/) | Passe die Darstellung von Objekten im Debugger individuell an – für .NET (DebuggerDisplay) und C++ (Natvis).                 |
| [Anheftbare Eigenschaften](https://devblogs.microsoft.com/visualstudio/pinnable-properties-debug-display-managed-objects-your-way/) | Wichtige Eigenschaften von Objekten im Debugger anheften, um sie immer sichtbar zu haben.                                    |
| [Text-Visualisierer](https://docs.microsoft.com/en-us/visualstudio/debugger/string-visualizer-dialog-box?view=vs-2019) | Zeigt String-Inhalte komfortabel als Text, HTML, XML, JSON an.                                                                 |
| [IEnumerable / DataTable Visualizer](https://devblogs.microsoft.com/visualstudio/datatable-visualizer-improvements/) | Visualisiere Listen, Tabellen und Datenstrukturen direkt im Debugger.                                                        |
| [Breakpoints-Fenster](https://learn.microsoft.com/en-us/visualstudio/debugger/using-breakpoints?view=vs-2022#BKMK_Specify_advanced_properties_of_a_breakpoint_) | Verwaltung und Konfiguration aller Breakpoints an zentraler Stelle.                                                          |
| [Bedingte Breakpoints](https://docs.microsoft.com/en-us/visualstudio/debugger/using-breakpoints?view=vs-2019#breakpoint-conditions) | Halte nur an, wenn eine definierte Bedingung erfüllt ist – z.B. bei bestimmten Variablenwerten.                              |
| [Tracepoints](https://docs.microsoft.com/en-us/visualstudio/debugger/using-breakpoints?view=vs-2019#BKMK_Print_to_the_Output_window_with_tracepoints) | Schreibe beim Erreichen einer Codezeile automatisch eine Nachricht ins Ausgabefenster, ohne den Debugger anzuhalten.         |
| [Abhängige Breakpoints](https://learn.microsoft.com/en-us/visualstudio/debugger/using-breakpoints?view=vs-2022#BKMK_set_a_dependent_breakpoint) | Ein Breakpoint wird nur aktiv, wenn zuvor ein anderer Breakpoint ausgelöst wurde.                                            |
| [Temporäre Breakpoints](https://learn.microsoft.com/en-us/visualstudio/debugger/using-breakpoints?view=vs-2022#BKMK_set_a_temporary_breakpoint) | Ein Breakpoint, der sich nach einmaligem Auslösen automatisch deaktiviert.                                                   |
| [Suche in Autos, Watch, Locals](https://devblogs.microsoft.com/visualstudio/enhanced-in-visual-studio-2019-search-for-objects-and-properties-in-the-watch-autos-and-locals-windows/) | Ermöglicht das schnelle Finden von Variablen und Eigenschaften in großen Objekten.                                           |
| [Data Breakpoints](https://devblogs.microsoft.com/visualstudio/break-when-value-changes-data-breakpoints-for-net-core-in-visual-studio-2019/) | Halte an, wenn sich der Wert einer bestimmten Variablen im Speicher ändert (für .NET Core und C++).                          |
| [Step Into Specific](https://www.poppastring.com/blog/debugger-tip-step-into-a-specific-method)         | Steige gezielt in eine von mehreren Methodenaufrufen in einer Codezeile ein.                                                 |
| [Funktions-Rückgabewerte anzeigen](https://docs.microsoft.com/en-us/visualstudio/debugger/debugger-tips-and-tricks?view=vs-2019#view-return-values-for-functions) | Zeigt im Callstack die Rückgabewerte von Methoden nach deren Ausführung an.                                                  |
| [Pseudovariablen](https://docs.microsoft.com/en-us/visualstudio/debugger/pseudovariables?view=vs-2019) | Spezielle Variablen wie `$exception` oder `$returnvalue` zur Anzeige von Debug-Informationen.                                |
| [Format-Spezifizierer C#](https://docs.microsoft.com/en-us/visualstudio/debugger/format-specifiers-in-csharp?view=vs-2019) | Steuere die Anzeige von Variablenwerten im Debugger durch spezielle Formatangaben.                                           |
| [Edit and Continue](https://docs.microsoft.com/en-us/visualstudio/debugger/edit-and-continue?view=vs-2019) | Ändere den Code während einer Debug-Session und setze die Ausführung ohne Neustart fort.                                     |
| [Call Stack](https://learn.microsoft.com/en-us/visualstudio/debugger/call-stack-window?view=vs-2022)   | Zeigt die Aufrufreihenfolge der Methoden und Funktionen, um den Ausführungspfad und Fehlerquellen nachzuvollziehen.             |
| [Immediate Window](https://learn.microsoft.com/en-us/visualstudio/debugger/immediate-window?view=vs-2022) | Ermöglicht das Auswerten von Ausdrücken, Ausführen von Methoden und dynamisches Ausprobieren von Code während des Debuggens. |
| [Diagnostics Tools Fenster](https://docs.microsoft.com/en-us/visualstudio/profiling/running-profiling-tools-with-or-without-the-debugger?view=vs-2019#BKMK_Quick_start__Collect_diagnostic_data) | Zeigt Ausnahmen, Speicherverbrauch, CPU-Auslastung und mehr während des Debuggens an.                                        |
| [Snapshots bei Ausnahmen](https://devblogs.microsoft.com/visualstudio/snapshots-on-exceptions-while-debugging-with-intellitrace/) | Erstellt automatisch Snapshots des Programmzustands bei Ausnahmen (IntelliTrace, Enterprise).                                |
| [First-Chance Exceptions](https://docs.microsoft.com/en-us/visualstudio/debugger/managing-exceptions-with-the-debugger?view=vs-2019#tell-the-debugger-to-break-when-an-exception-is-thrown) | Debugger hält bereits beim ersten Auftreten einer Ausnahme, noch bevor sie behandelt wird.                                    |
| [Exception Settings](https://learn.microsoft.com/en-us/visualstudio/debugger/using-the-exception-helper?view=vs-2022) | Steuerung, bei welchen Ausnahmen der Debugger anhalten soll (z.B. nur bei nicht behandelten oder bestimmten Exception-Typen).   |
| [Hot Reload](https://learn.microsoft.com/en-us/visualstudio/debugger/hot-reload?view=vs-2022)           | Code-Änderungen in laufenden Anwendungen anwenden, ohne Debugging zu unterbrechen oder Anwendung neu zu starten.                 |
| [Tasks-Fenster](https://learn.microsoft.com/en-us/visualstudio/debugger/using-the-tasks-window?view=vs-2022) | Zeigt laufende Tasks und deren Status in asynchronen Anwendungen an.                                                         |
| [Parallel Stacks Fenster](https://learn.microsoft.com/en-us/visualstudio/debugger/using-the-parallel-stacks-window) | Visualisiert parallele Threads und deren Callstacks, ideal für Multithreading-Debugging.                                     |
| [Parallel Stacks für Tasks](https://devblogs.microsoft.com/visualstudio/debugging-async-code-parallel-stacks-for-tasks/) | Erweitert das Parallel Stacks Fenster für asynchrone Tasks.                                                                  |
| [Rethrown Exceptions](https://devblogs.microsoft.com/visualstudio/exception-helper-rethrown-exceptions/) | Zeigt, wenn Ausnahmen erneut geworfen werden, und hilft, die Ursprungsstelle zu finden.                                      |
| [An Prozess anhängen](https://learn.microsoft.com/en-us/visualstudio/debugger/attach-to-running-processes-with-the-visual-studio-debugger?view=vs-2022) | Debugger an laufende Prozesse anhängen, z.B. für Dienste oder externe Anwendungen.                                           |
| [Remote Debugging](https://learn.microsoft.com/en-us/visualstudio/debugger/remote-debugging?view=vs-2022) | Debuggen von Anwendungen auf entfernten Rechnern über das Netzwerk mit Remote Debugger.                                         |
| [An Linux Docker Container anhängen](https://docs.microsoft.com/en-us/visualstudio/debugger/attach-to-running-processes-with-the-visual-studio-debugger?view=vs-2019#BKMK_Docker_Attach) | Debugging von .NET-Anwendungen direkt in laufenden Linux-Docker-Containern.                                                  |
| [Dump Files analysieren](https://learn.microsoft.com/en-us/visualstudio/debugger/debug-dump-files?view=vs-2022) | Analyse von Speicherabbilddateien (*.dmp) zur nachträglichen Fehlerdiagnose, auch wenn der Prozess nicht mehr läuft.            |
| [Source Link](https://github.com/dotnet/sourcelink/blob/master/README.md) | Automatischer Download von passendem Quellcode zu Assemblies für ein vollständiges Debugging-Erlebnis.                       |
| [Child Process Debugging Power Tool](https://marketplace.visualstudio.com/items?itemName=vsdbgplat.MicrosoftChildProcessDebuggingPowerTool) | Automatisches Debugging von Kindprozessen, die während des Debuggens gestartet werden.                                       |
| [Code Map](https://learn.microsoft.com/en-us/visualstudio/modeling/map-dependencies-across-your-solutions?view=vs-2022) | Visualisiert Abhängigkeitsgraphen und Aufrufbeziehungen im Projekt, unterstützt Navigation und Verständnis großer Codebasen.   |
| [Snapshot Debugger](https://devblogs.microsoft.com/visualstudio/snapshot-debugging-with-visual-studio-2017-now-ready-for-production/) | Debuggen von Live-Anwendungen (z.B. in Azure) anhand von Snapshots des Anwendungszustands.                                   |
| [Time-Travel Debugging](https://devblogs.microsoft.com/visualstudio/introducing-time-travel-debugging-for-visual-studio-enterprise-2019/) | Rückwärts-Debugging: Spule den Programmablauf zurück, um Fehlerquellen zu finden (nur Enterprise).                           |
| [Performance Profiler](https://learn.microsoft.com/en-us/visualstudio/profiling/?view=vs-2022) | Analysiere Speicher, CPU, Netzwerk und andere Ressourcen, um Performance-Engpässe zu identifizieren.                         |
| [IntelliTrace](https://learn.microsoft.com/en-us/visualstudio/debugger/intellitrace?view=vs-2022)      | Zeichnet automatisch wichtige Ereignisse und Zustände auf, ermöglicht „Zurückspulen“ im Debugging-Verlauf. Nur in Enterprise Edition. |
| [Live Unit Testing](https://learn.microsoft.com/en-us/visualstudio/test/live-unit-testing-start?view=vs-2022) | Führt Unit Tests automatisch während des Codierens aus und zeigt Ergebnisse sowie Testabdeckung direkt im Editor an.            |
